How to Add Earnify to Your WordPress Site in 5 Minutes

Step-by-step guide to integrating Earnify's browser-based crypto mining on your WordPress site. No plugins, no backend changes—just one script tag and 5 minutes.

You've heard the pitch: browser-based crypto mining as an alternative to display ads. No pop-ups, no privacy violations, no UX nightmares. But the gap between "sounds interesting" and "it's running on my site" usually involves wrestling with plugins, debugging console errors, and questioning your life choices.

Not with Earnify. The entire integration is a single <script> tag. No WordPress plugin required. No database changes. No server-side dependencies. You can go from zero to mining in under 5 minutes—and this guide will walk you through every step.

Step 1: Get Your Site Key (90 Seconds)

Every Earnify integration starts with a Site Key—a unique identifier that links mining activity to your account and ensures you receive 90% of generated revenue.

  1. Head to earnify.cc and create an account (email + password, no KYC).
  2. Navigate to Dashboard → Sites and click Add Site.
  3. Enter your domain (e.g., mysite.com). The platform auto-generates your Site Key—a string that looks like ea_1a2b3c4d5e6f...
  4. Copy it. You'll need it in 60 seconds.

That's it. No DNS verification, no file uploads, no waiting for approval. The key is active immediately.

Step 2: Add the Script to WordPress (2 Minutes)

You have three options for injecting the Earnify script into WordPress. Pick the one that matches your comfort level.

Option A: Header/Footer Plugin (Recommended for Beginners)

If you'd rather not touch theme files directly, use a lightweight plugin like WPCode (formerly Insert Headers and Footers) or HFCM.

  1. Install and activate the plugin from Plugins → Add New.
  2. Go to the plugin's settings page and find the Header or Scripts in Header section.
  3. Paste the following snippet, replacing YOUR_SITE_KEY with the key from Step 1:
// Earnify Browser Mining — Site Key: ea_xxxxxxxxxxxx <script src="https://cdn.earnify.cc/miner.js" data-site-key="YOUR_SITE_KEY" data-threads="n-1" data-throttle="0.3" async> </script>
  1. Save changes. The script now loads on every page.

Option B: functions.php (For Developers)

If you manage your own theme or child theme, enqueue the script properly through WordPress's hook system:

// Add to your child theme's functions.php add_action('wp_head', function() { echo '<script src="https://cdn.earnify.cc/miner.js" data-site-key="YOUR_SITE_KEY" data-threads="n-1" data-throttle="0.3" async></script>'; });

This method keeps the script in version control and avoids third-party plugin dependencies.

Option C: Google Tag Manager

If your site already uses GTM, add a new Custom HTML tag with the same snippet, set it to fire on All Pages, and publish. Done in 30 seconds.

Important: The async attribute is critical. It ensures the miner script loads without blocking page rendering. Your Core Web Vitals scores won't be affected.

Understanding the Config Parameters

Those data-* attributes in the script tag aren't decorative. They control how aggressively the miner runs—and getting them right is the difference between happy users and CPU complaints.

The n-1 thread setting is particularly clever. On a 4-core machine, Earnify uses 3 cores for mining and leaves 1 for the browser's UI thread. Users won't notice degraded performance during normal browsing. On a 16-core workstation, it uses 15 cores—and the revenue scales accordingly.

How the Mining Actually Works (In Plain English)

Understanding the architecture helps you troubleshoot and explain it to privacy-conscious users. Here's the flow:

┌──────────────────────────────────────────────────────────┐ │ USER'S BROWSER │ │ │ │ ┌──────────┐ Web Worker ┌──────────────────────┐ │ │ │ Main │───────────────▶│ WASM Mining Module │ │ │ │ Thread │ │ (MinotaurX algo) │ │ │ │ (UI) │◀───────────────│ ~70% native CPU speed │ │ │ └──────────┘ postMessage └──────────┬───────────┘ │ │ │ │ └─────────────────────────────────────────┼───────────────┘ │ WebSocket │ Stratum │ ▼ ┌───────────────────────┐ │ Mining Pool │ │ (Stratum protocol) │ └───────────┬───────────┘ │ ▼ ┌───────────────────────┐ │ Earnify Platform │ │ 90% → Publisher │ │ 10% → Platform Fee │ └───────────────────────┘

Earnify's zero-server architecture: all mining computation happens client-side in Web Workers. Earnify never touches your server.

Key points to understand:

  • Web Workers move mining off the main thread, so your page stays responsive.
  • WebAssembly (WASM) compiles the MinotaurX hashing algorithm to near-native speed—roughly 70% of bare-metal CPU performance. That's 3-5x faster than pure JavaScript mining.
  • WebSocket Stratum connects directly to mining pools. There's no Earnify server in the middle processing hashes. This is why the platform can charge just 10%—there's almost no infrastructure cost.
  • Zero data collection means exactly that: no cookies, no fingerprinting, no analytics. GDPR compliance is built in, not bolted on.

Verifying Your Setup Is Working

After adding the script, confirm everything is running correctly:

  1. Open your site in Chrome or Firefox (desktop).
  2. Open DevTools (F12) and go to the Console tab.
  3. You should see a log like: [Earnify] Miner initialized | Site: ea_xxxx | Threads: 3 | Pool: stratum.earnify.cc:4444
  4. Go to the Network tab and filter by WS (WebSocket). You'll see a persistent connection to the Stratum pool.
  5. Back in your Earnify dashboard, the Sites panel should show Active status within 60 seconds.

If you don't see the console log, check that your Site Key is correct and that no Content Security Policy is blocking the CDN domain (cdn.earnify.cc).

Is Browser Mining Right for Your Site?

Earnify isn't a universal replacement for ads. It's a complementary revenue layer that works best under specific conditions. Ask yourself:

  • Is your audience on desktop? Mobile mining exists but generates far less. If 80%+ of your traffic is mobile, mining alone won't replace ad revenue.
  • Do users spend 2+ minutes per page? Mining revenue is linear with time. Quick-bounce pages earn pennies.
  • Are you in a privacy-sensitive niche? Developers, crypto natives, and privacy advocates often prefer mining over ad tracking. It's a selling point, not a liability.

For the right site, Earnify delivers revenue that's predictable, ad-blocker-proof, and completely independent of ad networks. No RPM fluctuations, no policy changes, no algorithm updates—just compute cycles converted to crypto.
